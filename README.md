This is my year three computer science project which is focused on a study of various algorithms to be used to more efficently spell check

The system works as follows:

C_Program:
    - Holds all the working code for a spell checker system in C
    - to compile use: gcc -Wall -fopenmp -o <name-of-execuatable> Main.c ListGenerator.c SpellChecker.c
    -requires the files in the folder "corpus" to function

Python_Program:
    -Holds all the working code for spell checker system in Python3
    - Requires the following packages to be installed:
        *bs4
        *lxml
    -run with: python Main.py
    -requires the files in the folder "corpus" to function

proof-of-concept:
    -holds the working code for a protoype spell checker in C
    -to compile use: gcc -Wall -fopenmp -o <name-of-executable> main.c
    -doesn't require the files in the folder "corpus" to function, instead uses internal words_data folder

Suceessful Runs:
    -holds corpus word files generated by either of the main C / Python spell checker programs.
    -could be used with either system, though they would need to be dragged into their GeneratedFiles folder in order to be used

corpus:
    -The folder "Texts" contains the entire corpus that the main spell checker files use
    -the combined size of these corpora is around 4.6 GB, so downloading them would take time and space.
    -the texts are divided into 10 sub corpora, which each have varying numbers of word files of varying length as to test the spell checkers with wordcounts of different sizes.

project-writeup:
    -contains all files used for designing and then writing up the findings for the final report
