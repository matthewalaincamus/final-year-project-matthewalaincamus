This is my year three computer science project which is focused on a study of various algorithms to be used to more efficently spell check

The system works as follows:

##C_Program:
'''
    - Holds all the working code for a spell checker system in C
    - to compile use: gcc -Wall -o <name-of-execuatable> Main.c ListGenerator.c SpellChecker.c
    - requires the files in the folder "corpus" to function
    - to run the automated testing with misspelled words, type ! when given the option to (**note this could take up to an hour to complete**)
    - to navigate the menu, type the letters that are associated with a certain option
'''

##Python_Program:
'''
    - Holds all the working code for spell checker system in Python3
    - Requires the following packages to be installed:
    bs4
    lxml
    strsimpy
    timeit
    - run with: python Main.py
    - requires the files in the folder "corpus" to function
    - to run the automated testing with misspelled words, type ! when given the option to (**note this could take multiple hours to complete**)
    - to navigate the menu, type the letters that are associated with a certain option
'''

##AutomatedTesingResults
'''
    -Holds the results of the automated testing that both the Python and C programs
    -GraphGenerator.py uses matplotlib and numpy in order to generate graphs that are used in the writeup
    -requires the files generated from the automated testing functions of the C and Python programs
'''

##proof-of-concept:
'''
    -holds the working code for a protoype spell checker in C
    -to compile use: gcc -Wall -o <name-of-executable> main.c
    -doesn't require the files in the folder "corpus" to function, instead uses internal words_data folder
    - the words_alpha file was aquried from: https://github.com/dwyl/english-words
'''

##Suceessful Runs:
'''
    -holds corpus word files generated by either of the main C / Python spell checker programs.
    -could use the GeneratedWordFiles with either system, though they would need to be dragged into their GeneratedFiles folder in order to be used
    -the GeneratedResultFiles would need to be dragged into the AutomatedTestingResults folder in order to generate their graphs
'''

##corpus:
'''
    -The folder "Texts" contains the entire corpus that the main spell checker files use
    - this was got from: https://llds.ling-phil.ox.ac.uk/llds/xmlui/handle/20.500.14106/2554
    -the combined size of these corpora is around 4.6 GB, so downloading them would take time and space.
    -the texts are divided into 10 sub corpora, which each have varying numbers of word files of varying length as to test the spell checkers with wordcounts of different sizes.
    -For anyone wanting to use the system, corpora 'D' is the only small folder which should take less than a second to extract. All other corpora options will take a few minutes to compile given their size.
    - The folder "Words-Misspelled" contains 3 files of misspelled words for automated testing, however only the "aspell.dat.txt" file is used given the performance of the Python program (this was got from: https://www.dcs.bbk.ac.uk/~roger/corpora.html)
'''
